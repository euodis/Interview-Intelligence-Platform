# Расширенная история промптов / Detailed Prompts History — II-Platform

Этот документ содержит детализированную хронику запросов, которые сформировали архитектуру, логику и интерфейс платформы. Процесс разделен на 7 ключевых фаз, каждая из которых содержит глубокие технические требования.

---

## 1. Концепция и проектирование (Product Framing)

**Запрос:**
> Я хочу создать SaaS-платформу для автоматизации технического найма — **Interview Intelligence Platform**. 
> 
> **Проблема:** Интервьюеры пишут разрозненные заметки, оценки субъективны, а HR-менеджеру сложно собрать финальный вердикт, когда мнения команды расходятся.
> 
> **Задача:** Спроектируй систему, которая:
> 1. Позволяет создавать структурированные планы интервью для вакансий.
> 2. Собирает оценки (1-5) и текстовые сигналы по каждому блоку компетенций.
> 3. Использует ИИ для автоматического синтеза всех протоколов в единый **Candidate Summary** (Strengths, Risks, Verdict).
> 4. **Ключевая фича:** ИИ-модуль "Калибровки" (Calibration Engine). Если один эксперт поставил 2, а другой 5 за одно и то же — система должна не просто усреднить балл, а проанализировать причины конфликта через LLM и предложить план действий.
> 
> Дизайн: Современный, премиальный, в темных тонах с акцентами (glassmorphism), ориентированный на CTO и Lead-рекрутеров.

## 2. Архитектура и стек технологий (Architecture)

**Запрос:**
> Сгенерируй технологический стек и базовую структуру проекта:
> - **Frontend/Backend:** Next.js 16 (App Router). Обязательно используй Server Actions для всей бизнес-логики и Server Components для оптимизации LCP.
> - **База данных:** PostgreSQL + Prisma ORM. Нужно обеспечить строгую типизацию на уровне схемы.
> - **ИИ:** Интеграция с Claude-3.5/4.6 через OpenRouter. Все вызовы должны проходить через прослойку валидации **Zod**, чтобы исключить ошибки парсинга "галлюцинаций" ИИ.
> - **Безопасность:** NextAuth.js (Auth.js) с Credentials Provider для закрытого внутреннего контура.
> - **Инфраструктура:** Docker-контейнеризация с разделением на dev и production окружения. Обязательно используй multi-stage сборку для минимизации веса образа.

## 3. Проектирование интерфейсов и UX (UI Screens)

**Запрос 1 (Dashboard):**
> Создай основной дашборд вакансии. Требования к таблице кандидатов:
> - **Sticky Layout:** Первая колонка с именем кандидата должна быть закреплена при горизонтальном скролле (важно для широких таблиц с баллами).
> - **Full-row interaction:** Сделай всю строку кандидата кликабельной, а не только маленькую кнопку в конце.
> - **Визуальный статус:** Используй палитру "песочный/золотой" для статуса "ИНТЕРВЬЮ" и "приглушенный зеленый" для "ОЦЕНЕН".
> - Добавь фильтрацию и сортировку по среднему баллу.

**Запрос 2 (Comparison Tool):**
> Реализуй инструмент Side-by-Side сравнения. 
> - Пользователь может выбрать **максимум 2 кандидатов** (для фокуса).
> - По центру добавь блок "AI Verdict", где Claude анализирует обоих и предлагает "победителя" на основе требований вакансии.
> - Блоки компетенций должны выравниваться горизонтально для прямого сравнения оценок.

## 4. Логика ИИ и Промпт-инжиниринг (AI Logic)

**Запрос:**
> Напиши модуль `src/lib/ai.ts`. Мне нужны сложные системные промпты для Claude-3.5-Sonnet:
> 
> 1. **Calibration Prompt:** "Ты — эксперт по калибровке найма. Тебе даны полярные оценки. Проанализируй заметки интервьюеров, найди когнитивные искажения (например, один спрашивал теорию, другой практику) и предложи, нужно ли доп. интервью или просто синк команды."
> 2. **Summary Prompt:** "Синтезируй все протоколы сессий. Выдели 'Notable Evidence' — прямые цитаты или факты из заметок, которые подтверждают оценку."
> 3. **Validation:** Используй `response_format: { type: "json_object" }`. Все строки в JSON должны быть на **русском языке**. Обязательно оберни парсинг в блок `try-catch` с возвратом локальных моков при сбое API.

## 5. Моделирование данных и сиды (Data Setup)

**Запрос:**
> Спроектируй схему `schema.prisma`. Важные сущности:
> - **Vacancy:** Название, уровень, competencies (JSON или связанные записи).
> - **Application:** Связь Candidate <-> Vacancy.
> - **InterviewSession:** Конкретная встреча с полем `notes` (текст от эксперта).
> - **BlockEvaluation:** Оценка (Integer) за конкретную компетенцию в рамках сессии.
> 
> Создай мощный `prisma/seed.ts`:
> - Добавь вакансию "Frontend Lead".
> - Создай 3 кандидата. Один из них должен иметь "конфликтные" оценки (например, 2 от одного эксперта и 5 от другого за Архитектуру), чтобы я мог протестировать Calibration Engine.

## 6. Шлифовка и отказоустойчивость (Polish & Resilience)

**Запрос:**
> **Критическая правка:** Иногда LLM возвращает JSON, обернутый в маркдаун \`\`\`json ... \`\`\`. Это ломает `JSON.parse`. 
> - Напиши защитную функцию `cleanJsonString`, которая использует регулярные выражения для извлечения сырого JSON. 
> - Добавь эту очистку во все AI-экшены.
> - Проверь, что при нажатии "Сгенерировать план" кнопка переходит в состояние Loading и блокируется до завершения процесса.
> - Убедись, что все AI-ответы строго локализованы на русский язык.

## 7. Инфраструктура и Production Setup (Deployment)

**Запрос:**
> Подготовь проект к деплою на внешний сервер:
> 1. **Dockerfile:** Используй `node:20-slim`. Внедри кэширование BuildKit для `node_modules` и `.next/cache`, чтобы сборка на CI не занимала 20 минут.
> 2. **Порты:** У меня 3000 порт занят. Переведи всё приложение на порт **3003** (включая EXPOSE и ENV PORT внутри докера).
> 3. **Reverse Proxy:** Напиши пример конфига Nginx, который проксирует трафик с IP сервера на 3003-й порт контейнера с заголовками `Upgrade` для работы веб-сокетов Next.js.
> 4. **Guide:** Составь пошаговый `deployment_guide.md` для запуска с нуля через `docker compose -f docker-compose.prod.yml up -d`.
